## �� Branching Strategy
We use a **feature branch workflow**. Never commit directly to `main`.
### Branch naming convention
| Prefix      | Use case                      | Example                       |
|-------------|-------------------------------|-------------------------------|
| `feature/`  | New features or functionality | `feature/product-listing-page`| 
| `fix/`      | Bug fixes                     | `fix/cart-quantity-update`    |
| `docs/`     | Documentation only            | `docs/api-endpoints`          |
